# Value Investment Agent — 需求文档（草案）

> **版本**：0.2 草案  
> **依据**：当前仓库实现、`docs/architecture.md`、交互讨论（含 moutai 四线实验）、以及你提供的 research proposal（PDF 未纳入本仓库；若与 PDF 条款冲突，以你确认的修订版为准）。  
> **维护**：本文档随讨论与调测持续修订。

---

## 1. 项目目标

构建一套 **神经符号（neuro-symbolic）价值投资代理** 系统，核心能力包括：

1. **符号侧 Fi**：基于可解释估值核（DCF、P/S、PB-ROE 等）得到内在价值或相对价值序列。  
2. **神经侧 Fm**：在 Fi 护栏下学习市场定价偏差、动量、风险溢价等。  
3. **参数合成器（Parameter Synthesizer）**：将时间序列与因子映射到估值核输入（moutai 简化实验可绕过，仅用规则调制）。  
4. **ViAgent（主 Agent）**：总编排、整体流程与多轮人机交互（见 §3.6）。  
5. **Qualitative Factors Subagent**：专职定性因子的抽取、重大新闻后的再评估与打分（见 §3.6）；全系统 **仅保留这两个 Agent**，其余为普通代码模块。  
6. **数据与因子分层**：**`data/`** 存 **抓取落地**（个股 `data/{symbol}/raw/`，全市场宏观 **`data/macro/raw/`**）；**`factors/`** 存 **由 data 清洗或计算得到的因子时间序列**（个股 `factors/{symbol}/`，宏观 **`factors/macro/series/`**）。宏观指标（美联邦基金利率、中国 LPR、中美国债收益率、美 PCE、中国侧通胀对标序列等）**原始文件**在 `data/macro/raw/`，**标准化序列**在 `factors/macro/series/`；与 `cola`、`moutai` 等**并列**，不按个股分目录。标的 slug 统一小写（见 `config/symbols.py`）。

**成功标准（初稿）**：可追溯的数据血缘、可复现实验（含 moutai 五年四线看板）、因子与估值假设可审计；后续可扩展回测指标与 Fi/Fm 稳定性评估。

## 3. 功能需求

### 3.1 数据采集与存储

- **行情**：日线收盘价等（如 Yahoo，标的映射可配置）。  
- **财报**：利润表/资产负债表/现金流量表或聚合指标（本地 CSV 优先，失败时可回退 API）。  
- **文本与事件**：新闻摘要、公告要点（如 `news_digest.csv`）；可选追加第三方标题。  
- **输出**：清洗后的时间序列落盘，路径与命名规范见设计文档；**禁止**在目录名中暴露 Yahoo ticker。

### 3.2 因子体系（全局池 + 个股子集）

**需求 FR-FACTOR-1**：系统必须维护 **两类因子定义的全局注册表**（定量 / 定性），以表格或机器可读清单形式版本化管理，**随调测可频繁增删改**，并保留变更说明或版本号。

**需求 FR-FACTOR-2**：表中因子对 **所有股票** 在定义上均可用，构成 **大的因子池**；池子会随时间 **持续扩充**。

**需求 FR-FACTOR-3**：对 **每个个股（symbol）**，仅启用 **对该标的分析有意义** 的因子子集，并为每个启用因子配置 **权重**（及可选：启用/禁用、数据来源覆盖规则）。同一因子在不同标的上的权重可以不同。

**需求 FR-FACTOR-4**：定量因子命名采用 **业界通用英文标识**（见第 4 节初始表）；文档中可同时给出中文释义与常见报表对应关系。

**需求 FR-FACTOR-5**：定性因子为 **结构化评分维度**；**全系统统一量纲**（见 §3.2.1）。支持 LLM 打分或规则启发式；全局池定义与设计文档一致，个股层可选择子集与权重。

**需求 FR-FACTOR-6（Fi 与权重）**：个股因子 **权重参与 Fi 的计算**，其中 **定性因子权重为关键输入**。每个标的因子权重 **各不相同**，反映个股特色。权重 **建议** 由后续在个股数据上 **训练** 得到；同时允许人工根据先验为每个标的设置 **初始权重** `w_initial`。训练过程中 **不得修改** `w_initial`，**只更新** **调整权重** `w_adjustment`。**参与 Fi 数值计算的权重一律为 `w_adjustment`**，`w_initial` 作为先验锚定与审计追溯，**不直接进入 Fi 计算公式**（若未来需要「初始 + 调整」联合进 Fi，须单独变更需求与版本）。

### 3.2.1 定性因子评分量纲

- **范围**：**0–10 分**（全系统统一，不因因子而异）。  
- **步长**：**0.5 分**（允许 **5.5**、**7.0** 等）。  
- **存储与校验**：实现中按 **0.1 或更细内部精度** 存储亦可，但 **对外与 LLM 约定** 为 0.5 步长；校验时须落在 `0, 0.5, 1.0, …, 10.0`。

### 3.3 全局指标定义与开关（D/E、FCF 等）

**需求 FR-METRIC-1**：`debt_to_equity`、`free_cash_flow` 等定量指标的 **精确定义式**（分子分母、合并口径、期末/平均等）不得在代码里写死多处，须维护 **一份全局可配置参数表**（见设计文档 §3.1）。

**需求 FR-METRIC-2**：当同一指标存在多种定义（例如 FCF 的 **FCFF** vs **FCFE**），在参数表中 **枚举可选 variant**，并 **指定当前默认** `default_variant_id`；运行与回测以该默认为准，切换 variant 时须在元数据中记录。

**需求 FR-METRIC-3**：该全局表 **可扩展**，用于承载与本版本相关的其他开关，例如 **本版本暂不计算 Fm**（`compute_fm: false` 等），便于后续无改代码结构即可演进。

### 3.4 估值与展示

- 支持至少一种 **端到端** 流程：原始数据 → 因子/调制 → Fi 序列 → 与市场价/均线同图展示（moutai 实验为参考实现：收盘价、MA120、Fi 纯 DCF、Fi+定性调制）。  
- 输出物包括图表、CSV/JSON 元数据，便于审计与复现。

### 3.5 LLM 与可配置性

- 支持 `auto` / `gemini` / `openai` / `mock`；密钥通过环境变量配置；无密钥时回退启发式（行为以设计文档为准）。  
- 定性打分所依赖的文本上下文应可追溯（如 digest 路径、摘要片段长度上限）。

### 3.6 Agent 架构（固定为 2 个）

| Agent | 职责 |
|-------|------|
| **主 Agent（ViAgent）** | 端到端流程编排、工具/模块调用、**多轮用户交互**、会话状态与任务路由。 |
| **子 Agent（Qualitative Factors Subagent）** | **仅**负责与 **定性因子** 相关：文本与重大新闻输入下的因子抽取、更新与 **0–10（0.5 步长）** 打分；输出结构化结果供主 Agent 写入 `factors/` 并参与 Fi。 |

**需求 FR-AGENT-1**：重大新闻导致定性因子变化时，由 **Qualitative Factors Subagent** 计算/更新得分，而非在主 Agent 内嵌一整套定性逻辑。  
**需求 FR-AGENT-2**：不新增第三类「Agent」；采集、回测、估值核、Fm 训练等均为 **普通 Python 模块**，由主 Agent 调用。

### 3.7 非功能需求

- **可维护性**：因子表、个股因子配置与代码逻辑分离，便于非开发人员改表后再跑。  
- **可扩展性**：新增标的、新增因子、新增估值核不破坏现有 slug 与目录约定。  
- **一致性**：与 `docs/architecture.md` 中的目录与模块划分一致；冲突时以经评审的更新为准。

## 4. 全局因子池 — 初始定义（v0）

以下为 **第一版** 常见条目；**非穷尽**。后续在注册表中增加 `factor_id`、数据类型、频率、报表来源、备注与版本字段（详见设计文档）。

### 4.1 定量因子（Quantitative）— 初始条目

| factor_id | 标准名称（EN） | 中文释义 | 备注 |
|-----------|----------------|----------|------|
| `net_income` | Net income | 净利润 / 净收益 | 利润表「底线」；与 `net_profit` 在多数语境下 **同义**，IFRS 常用 *profit for the period*；美股报表常用 **Net income**。建模时 **二选一作为主因子** 即可，避免重复计权。 |
| `net_profit_attributable` | Net profit attributable to owners | 归母净利润 | A 股财报常见披露口径；与合并报表中归属于母公司所有者的净利润一致。若已用 `net_income` 覆盖同一口径，可不再单独启用。 |
| `debt_to_equity` | Debt-to-equity ratio (D/E) | 负债权益比（常见：**总债务 / 股东权益**） | 与 **资产负债率** 不同；具体分子分母以全局参数表 **variant** 为准（见 §3.3）。 |
| `free_cash_flow` | Free cash flow (FCF) | 自由现金流 | 需声明 **FCFF**（企业）还是 **FCFE**（股权）；与 DCF 口径一致。 |
| `capex` | Capital expenditures (CapEx) | 资本性支出 | 现金流量表投资活动；常结合折旧用于衡量再投资强度。 |

**说明**：你原先列的「net income 与 net profit」在英文财报中常混用；上表将 **归母净利润** 单独列出以便 A 股场景，避免与合并净收入口径混淆。

### 4.2 定性因子（Qualitative）— 初始条目

以下 **10 项** 为全局池初稿；每项应有稳定 `factor_id`（蛇形命名）与中文名称，便于 LLM prompt 与配置引用。**打分一律遵循 §3.2.1（0–10，步长 0.5）**。

| factor_id | 英文名称 | 中文释义 |
|-----------|----------|----------|
| `market_potential` | Market potential | 市场潜力 / 空间（含 TAM 与渗透） |
| `pricing_power` | Pricing power | 提价权 / 定价权 |
| `branding_power` | Brand strength | 品牌力（原 branding power，标准表述常用 brand strength） |
| `management_quality` | Management quality & incentives | 管理层能力与激励相容（原 management stimulus，改为更通用表述） |
| `strategy_soundness` | Strategic soundness | 战略合理性 / 执行度（原 strategy rightness） |
| `competitive_position` | Competitive position | 竞争地位 |
| `industry_position` | Industry position | 产业地位 / 产业链位置 |
| `shareholder_reward` | Shareholder returns policy | 股东回报（分红、回购、再投资平衡） |
| `balance_sheet_health` | Balance sheet health | 资产负债表健康度 |
| `cash_flow_health` | Cash flow health | 现金流健康度 / 盈利质量 |

## 5. 个股因子配置（需求摘要）

- 每个 `symbol` 对应一份 **因子子集 + 双轨权重**（见设计文档）：`w_initial`（人工先验，训练期不变）、`w_adjustment`（可训练，**Fi 仅使用此项**）。  
- 权重之和不一定为 1（由上层合成规则决定）；若需要归一化，应在设计文档中固定一种归一化方式。  
- 配置变更应可版本化，便于回测对比。

---

## 6. 约束与假设

- moutai 实验为 **简化打穿**：可无 Parameter Synthesizer，仅用规则对 growth/WACC 做定性调制。  
- 全系统 Fi/Fm 与 Synthesizer 的完整联动以 research proposal 与后续设计为准。  
- 中国大陆与美股财报口径差异通过因子元数据与数据源标注区分。
